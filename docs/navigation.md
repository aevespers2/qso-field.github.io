Every repository should expose:

- overview
- architecture
- roadmap
- examples
- API or schema references
- contribution guide

## Cross-Linking Rules

Documents should link to:

- related repositories
- related specifications
- security assumptions
- future work

## Goal

A new contributor should understand the ecosystem in less than one hour of reading.
